# 15. Краткий сценарий показа на защите

## Порядок показа
1. Открываешь корень репозитория.
2. Показываешь `assignments/` и `resources/screenshots/`, чтобы было видно, что решение привязано к исходным заданиям.
3. Открываешь `CMakeLists.txt` и объясняешь структуру.
4. Собираешь проект.
5. Запускаешь `lab1_demo`, `lab2_demo`, `lab3_demo`.
6. Запускаешь `ctest`.
7. Открываешь `.github/workflows/ci.yml`.
8. Показываешь ветки и Pull Request на GitHub.
9. При необходимости показываешь `debug_examples/`.

## Фразы, которые полезно проговорить
- «Для СТРПО использую одну кодовую базу вместе с СД, потому что это прямо разрешено в `labs.md`.»
- «Сборка вынесена в CMake, исходники перечислены явно, `file(GLOB ...)` не использую.»
- «Внутри проекта есть статическая библиотека `ds_core`, а внешний бинарный артефакт подключается через `find_library` из заранее заданного каталога.»
- «Тесты регистрируются через `add_test` и запускаются через `ctest`, а GitHub Actions гоняет Debug/Release и GCC/MSVC.»
- «Оригинальные задания я сохранил отдельно, чтобы на защите было видно, какое место кода что именно закрывает.»

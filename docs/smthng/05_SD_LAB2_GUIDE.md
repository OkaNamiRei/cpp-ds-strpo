# 05. СД Лаба 2 — подробный разбор

## Где смотреть оригинал

`assignments/sd/original/lab2_original.cpp`

## Что закрывает лабораторная

### `BaseFile`
Файлы:
- `include/ds/base_file.hpp`
- `src/base_file.cpp`

Показывает:
- владение ресурсом `FILE*`;
- `is_open`, `can_read`, `can_write`;
- `read_raw`, `write_raw`, `tell`, `seek`;
- удалённые copy-операции;
- move-операции;
- виртуальные `read` и `write`.

### `Base32File`
Файлы:
- `include/ds/base32_file.hpp`
- `src/base32_file.cpp`

Показывает:
- наследование от `BaseFile`;
- переопределение `read`/`write`;
- кодирование/декодирование данных;
- корректное чтение маленькими порциями благодаря внутреннему кэшу.

### `RleFile`
Файлы:
- `include/ds/rle_file.hpp`
- `src/rle_file.cpp`

Показывает:
- наследование;
- сжатие и распаковку RLE;
- последовательное чтение и запись без потери состояния.

### `IFile`
Файл:
- `include/ds/ifile.hpp`

Идея:
- общий интерфейс файла;
- нужен для позднего связывания и композиции.

### `Base32File2`, `RleFile2`
Файлы:
- `include/ds/composed_files.hpp`
- `src/composed_files.cpp`

Идея:
- композиция вместо жёсткого наследования;
- класс оборачивает другой `IFile`.

## Что объяснять

### Как наследовать, чтобы методы базового класса были доступны
`public`-наследованием.

### Что такое раннее связывание
Когда выбор функции решается по статическому типу на этапе компиляции.

### Что такое позднее связывание
Когда при вызове через базовый тип выбирается версия метода по реальному типу объекта во время выполнения.

### Почему нужен виртуальный деструктор
Чтобы удаление через `BaseFile*` корректно вызывало и деструктор наследника, и деструктор базового класса.

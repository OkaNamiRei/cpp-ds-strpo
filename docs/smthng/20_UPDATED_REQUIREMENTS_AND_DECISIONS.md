# 20. Новые вводные и почему проект устроен именно так

## Какие источники требований учтены

1. Оригинальные `lab1.cpp`, `lab2.cpp`, `lab3.cpp` по СД.
2. Старый `labs.md` по СТРПО.
3. Новый `labs_final_complete.md`, который преподаватель подменил позже.
4. SSH-гайд.
5. Скриншоты сайта и переписки.
6. Устные уточнения по переписке:
   - SSH желателен, но не обязателен;
   - первую лабу по СД тоже можно собирать через CMake;
   - для отладочной лабы допустимо использовать Visual Studio.

## Ключевые решения

### Почему оставлен один репозиторий
Потому что актуальные требования по СТРПО прямо разрешают работать параллельно с лабораторными по СД на одной кодовой базе.

### Почему сохранены и старая, и новая версия требований
Чтобы на защите было видно, что проект не фантазийный, а адаптирован к реальному обновлению задания.

### Почему build-папки убраны из финального архива
Потому что их не надо хранить в репозитории. Они только мусорят и мешают показывать структуру.

### Почему СТРПО-6 только подготовлена
Потому что без курсовой её нельзя честно закрыть. Притворяться тут бессмысленно: преподаватели обычно замечают такие чудеса.

### Почему СТРПО-7 лежит отдельно
Потому что это независимая необязательная лабораторная и ей не нужен весь основной проект.

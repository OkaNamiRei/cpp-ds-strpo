# 04. СД Лаба 1 — подробный разбор

## Где смотреть оригинал

`assignments/sd/original/lab1_original.cpp`

## Что закрывает эта лабораторная

### 1. `Rect`
Файлы:
- `include/ds/rect.hpp`
- `src/rect.cpp`

Что демонстрируется:
- конструктор по умолчанию;
- параметризованный конструктор;
- конструктор копирования;
- move-конструктор как бонус;
- деструктор;
- приватные поля;
- get/set методы;
- перегруженные `inflate`;
- `move(dx, dy)` с параметрами по умолчанию;
- `bounding_rect`;
- свойства ширины/высоты/площади.

### 2. `Barrel`
Файлы:
- `include/ds/barrel.hpp`
- `src/barrel.cpp`

Идея:
- объект моделирует ёмкость с водой/спиртом;
- доля спирта хранится в обычных десятичных числах;
- `%` специально не нужен, потому что преподаватель просил без него.

### 3. `Matrix`
Файлы:
- `include/ds/matrix.hpp`
- `src/matrix.cpp`

Идея:
- матрица хранится **без динамической памяти**;
- используется фиксированный массив `data_[16][16]`;
- это совпадает с пожеланием преподавателя «желательно не использовать динамическую память».

### 4. `MyString`
Файлы:
- `include/ds/my_string.hpp`
- `src/my_string.cpp`

Идея:
- класс владеет `char*`;
- сам выделяет и освобождает память;
- подходит для объяснения copy / move / operator=.

### 5. `TextWrapper`
Файлы:
- `include/ds/text_wrapper.hpp`
- `src/text_wrapper.cpp`

Идея:
- это пример «объекта-алгоритма»;
- внутри хранит текст и ширину;
- умеет считать строки и печатать переносы.

## Что важно проговорить на защите

### Почему `Rect rect2 = rect1;` — это копирование
Потому что создаётся **новый** объект `rect2`, который инициализируется объектом `rect1`. Это не оператор присваивания уже существующему объекту, а **копирующая инициализация**.

### Что такое инкапсуляция
Скрытие реализации. Пользователь класса работает через публичный интерфейс, а не ковыряет внутренности напрямую.

### Чем функция-член отличается от обычной функции
- вызывается у объекта;
- имеет скрытый параметр `this`;
- имеет доступ к `private`-полям.

### Когда вызывается деструктор
Когда объект выходит из область видимости или удаляется через `delete`.

### Почему Matrix без динамической памяти
Потому что это отдельно рекомендовал преподаватель. Так проще доказать корректность хранения и не плодить лишнюю память там, где лабораторная про другое.

# СД-2: файловые абстракции, Base32/RLE, композиция

## Быстрый запуск

```powershell
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
.\build\Debug\lab2_demo.exe
```

## Что здесь реализовано

- `IFile`: общий интерфейс файла.
- `BaseFile`: низкоуровневая работа с файлом.
- `Base32File` и `RleFile`: обёртки для кодирования.
- Композиция обёрток через одинаковый интерфейс.

## Что открывать на защите

1. `assignment_original/lab2_original.cpp`.
2. `include/ds/ifile.hpp` — общий интерфейс.
3. `base_file.*` — реальная работа с `FILE*`.
4. `base32_file.*`, `rle_file.*` — обёртки.
5. `composed_files.*` — цепочка декораторов.
6. `apps/lab2_demo.cpp` и `tests/`.

## Что отвечать

- `IFile` нужен, чтобы разные реализации файла использовать одинаково.
- Декоратор — объект с тем же интерфейсом, который добавляет поведение поверх другого объекта.
- `virtual` нужен, чтобы метод выбирался по реальному типу объекта.
- Move нужен, чтобы передавать владение файловым ресурсом без копирования.

# Инструкция по генерации SSH ключей и привязке к GitHub

## Зачем нужен SSH
SSH позволяет работать с GitHub без ввода логина и пароля при каждом `git push` и `git pull`.

---

## 1 Создание SSH-ключа

Во **всех системах** используется одна команда:

```bash
ssh-keygen -t ed25519
```

- на все вопросы можно просто нажимать **Enter**, но также можно установить дополнительный пароль, который будет запрашиваться каждый раз при использовании ключа

После этого будут созданы два файла:
- **приватный ключ** — `id_ed25519` (никому не показывать)
- **публичный ключ** — `id_ed25519.pub` (его добавляют в GitHub)

---

## 2️ Куда положить ключи

### Windows (Git Bash / PowerShell)

Ключи **обязаны лежать здесь**:

```
C:\Users\<ИмяПользователя>\.ssh\
```

> ❗ Не класть ключи в папку проекта  
> ❗ Не класть в `Program Files`  

---

### Unix (Ubuntu и другие дистрибутивы)

Ключи должны лежать **здесь**:

```
/home/<username>/.ssh/
```

> Ключ из Windows **не появляется автоматически в WSL**  
> Его нужно **скопировать вручную** из Windows.

---

## 3️ Добавление ключа в GitHub

1. Зайти на https://github.com  
2. Открыть: `Settings → SSH and GPG keys`
3. Нажать **New SSH key**
4. Вставить содержимое файла `id_ed25519.pub`
5. Сохранить

---

## 4️ Проверка работы SSH

В терминале выполнить:

```bash
ssh -T git@github.com
```

Если появляется сообщение:
```
Hi username! You've successfully authenticated...
```

то все сделано правильно

---

## 5️ git clone

- Для клонирования репозиториев использовать **SSH**, а не HTTPS:
  ```bash
  git clone git@github.com:user/repo.git
  ```

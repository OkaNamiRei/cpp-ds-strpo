# СТРПО-5: GitHub Actions + CTest

Здесь лежит минимальная кодовая база `project/` и workflow `.github/workflows/ci.yml`.

## Что показать
1. `.github/workflows/ci.yml`.
2. На GitHub вкладку Actions.
3. Успешные job'ы сборки и тестов.

## Что сказать
CI автоматически собирает проект и запускает тесты при push/PR. Это защищает `main` от очевидно сломанной сборки. Машина проверяет, что человек снова не сломал машину. Романтика.
